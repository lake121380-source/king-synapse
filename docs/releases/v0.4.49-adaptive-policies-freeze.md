# King Recall v0.4.49 - Adaptive Policies Freeze

## Status

Adaptive Policies is now considered **contract-frozen**.

This release closes P4.5 and closes Phase 4. It defines a dedicated decision layer above the frozen Adaptive Memory execution chains. It does not implement any concrete policy algorithms, does not change Recall, Memory Evolution, or Store Integration contracts, and does not introduce durable writes.

## Decision Model

```text
PolicyRequest
  -> AdaptivePolicyEngine
  -> PolicyReport
  -> PolicySink
```

`PolicyDecision::{Execute, Skip, Delay}` is the canonical output. Policies never mutate memory, never touch Store, Recall, or LLMs, and never perform execution.

## Freeze Rule

Future policy behavior must extend these interfaces rather than change them. Concrete policy algorithms (forgetting curves, reinforcement thresholds, merge scoring, LLM-driven decisions) will be introduced through follow-up RFCs after v0.5.0.

Breaking changes require ADR approval and a new architecture milestone.

## Benchmarks

- `multihop` remains `Recall@10 = 0.600`.
